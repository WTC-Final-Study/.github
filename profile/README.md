# 🧩 우테코 8기 스터디

### 스터디 방식
1. 미션 레포지토리의 main 브랜치를 clone한다.
     - `git clone <repo_url>`
2. 브랜치 생성 후 미션 구현을 시작한다.
     - `git checkout -b {github_id}/{n}`
5. 구현 완료 후 main 브랜치로 Pull Request를 보낸다.

### 🌿 브랜치 규칙

각자 풀이 기록을 명확하게 남기기 위해서 아래와 같은 브랜치 네이밍 규칙 사용
```
{github_id}/{n}
```
예시
- baekcci/1 — baekcci의 1차 풀이
- baekcci/2 — baekcci의 2차 풀이

### 🔀 PR(Pull Request) 규칙
PR 제목 형식:
```
{github_id}: {n}차 구현 완료
```

PR 템플릿
```
### 🕒 구현 시간
- 시작: YYYY-MM-DD HH:MM (선택)
- 종료: YYYY-MM-DD HH:MM (선택)
- 총 소요 시간: HH시간 MM분 (필수)

### 질문 또는 피드백 요청 (선택)
- 리뷰 받고 싶은 부분이 있다면 작성
```
